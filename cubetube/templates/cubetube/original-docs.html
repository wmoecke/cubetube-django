L3D Cube Specifications

The Kit

If you’re here in the Documentation section, you’re likely one of the first owners of a L3D Cube (8x8x8 edition) -- and this section of the site is intended to help you create and share applications on your Cubes.

If you’ve any questions that aren’t addressed here, head on over to the Forum or email Shawn & Alex at info@lookingglassfactory.com.

8x8x8 Cube Specifications

There are two types of L3D Cubes so far: the 8x8x8 edition with 512 RGB LEDs and the 16x16x16 edition with 4096 RGB LEDs. This documentation section currently only addresses the 8x8x8 edition and will be expanded to include the larger 16x16x16 edition in Spring 2015.

Each L3D Cube 8x8x8 edition comes with 512 RGB LEDs mounted in an array, powered over USB, and connected to WiFi. The brain and WiFi connection center of the Cubes is a Spark Core (soon to be a Spark Photon), which can be programmed like an Arduino. There’s also a built in microphone and accelerometer in each Cube.

Each Cube can download new apps from CubeTube with a click of a mouse or finger -- no programming needed. But of course for coders out there, what you create for your Cube can now be shared on CubeTube with other Cube owners....including those that own L3D Cubes and other types of 3D LED cubes.

LEDs: 512 RGB surface-mounted LEDs.
The specific type of LED used is a WS2812B, made by WorldSemi. There are 8 LEDs to each LED reed, and each LED reed is identical, connected into a custom base PCB.

Power: 5V 1A USB power can power most applications on your Cube. We’ve also included a 2A adapter. 100VAC - 220VAC input for higher power demand apps. And there’s a bypass around the 1.7A current-limit we’ve built into the Cubes too, for ridiculously bright applications.

Casing: Laser-cut acrylic housing

Powering Up

The power

Each L3D Cube comes with a 3-ft microUSB cable and 2A adapter. 5V, 500mA USB power can power the Default Demos and many of the apps you’ll find on CubeTube - so your Cube can run off your computer, wall wart, or a battery pack. But we’ve also included the 2A adapter with 100VAC - 220VAC input for more intense visualizations (you will need to supply your own prong adapter if outside the US).

Note 1:
There is a 1.7A current limit built into the baseboard of the 8x8x8 Cubes.

Note 2:
There is a bypass built into the baseboard of the 8x8x8 L3D Cubes which allows advanced users to get around the current limit for super bright cubing. Some of these modifications are popping up on the Forum, check them out!

Baseboard of the 8x8x8 Cubes.

Playing Default Demos

Demos

The 8x8x8 L3D Cubes come pre-loaded with four demos:

  • Demo 1. Fireworks
  • Demo 2. Zplasma
  • Demo 3. Squarral
  • Demo 4. Purple Rain (sound reactive)

The four demos will loop forever by default. You can stop the loop and play just one selected demo by tilting left, right, and forward.

The loop can be restarted by reconnecting to power.

Connecting to WiFi

If you already have a L3D Cube, you’re in the very earliest adopter group - a pioneer, you might say. You can connect your Cube to WiFi through the Spark App. These are instructions for how to do so come directly from spark.io.

Getting into Listening Mode

Each Cube is controlled by a Spark Core (soon to be a Spark Photon). This is the small circuit board that you plug your USB cable into for power.

When you plug in your 8x8x8 L3D Cube for the first time and switch the slider-switch at the back of your Cube to "Online", you should see the Spark Core start to blink a blue light. If not, then press down the "Join WiFi" button next to the Spark Core.

Once your Spark Core on the underside of your Cube is blinking a blue light, it’s ready to be connected to your local WiFi network via the Spark App.

Using the Spark App for Connecting to WiFi

Download the Spark App to your phone from the App Store or Google Play . (If you don’t have a phone that can download apps, no problem...you can still get your Cube online, it’s just a little more involved of a process to Connect over USB)

Once you’ve downloaded the Spark Core app from the App Store or Google Play, you should create an account. Afterwards, you’ll be asked to connect your Core,

Spark App Login screen on iOS.

Your phone must be connected to the Wi-Fi network that you want to connect the Core to. Wi-Fi Hotspots generated from the phone you are running this app on typically will yield an error claiming there is no Wi-Fi available.

In this case, please try to Connect over USB and enter your Hotspot credentials manually.

When connected to Wi-Fi, the app will automatically fill the SSID field with the name of the network that your phone is connected to. Enter your Wi-Fi password and hit connect.

This connection process can take up to a minute, so be patient. The closer your phone is to your Spark Core, the faster it will connect. Once the Core hears the signal, it will go through the following sequence of lights:

  • Solid blue: Credentials captured
  • Flashing green: Connecting to Wi-Fi network
  • Flashing cyan: Connecting to Spark Cloud
  • Breathing cyan: Connected to Spark Cloud

Once the Spark Core has connected, your phone will "claim" the Core and attach it to your account. Then you’ll get to name your Core. If you’re uncertain, you can confirm that the claim process was successful by logging into the Spark Web IDE and clicking the "Cores" icon at the bottom of the page. Is your Core listed? Great! The world is perfect.

The Core MUST be online (breathing cyan) in order for the claiming process to work. If the Spark Core has been claimed by someone else, the app won’t recognize it.

If you are connecting multiple Cores, you’ll go through this naming process for each Core. You’ll know which one is which by the rainbow signal.

Claiming your Core

Once your Cube’s Spark Core is connected to WiFi via the Spark App, it needs to be claimed in order to be associated with your account. This is what lets you control your Cube and keeps anyone else from doing so.

If you use the mobile app to set up your Core, it should claim it automatically. However if you connect your Core over USB, or if the claiming process is unsuccessful, you can claim it manually.

The easiest way to manually claim a Core over USB is to use the Spark Command Line Interface. Once you have this installed, you can simply type spark setup and follow the instructions.

Alternatively, if you have troubles installing the command line tool, you can get the Core’s ID over serial and claim it via the build site. You can do this by opening a Serial connection to the Core and pressing the i key (see the above instuctions for connecting over USB). It should show you a number like this:

# Example Core ID55ff68064989495329092587

Then open up Spark Build and click the ’Cores’ icon. Click the button that says ’Add a Core’, and enter your ID in the text box.

Switching to New WiFi Locations

When you move to a new WiFi locations, you can get your Cube’s Spark Core back into Listening Mode by clicking the big round "Connect to WiFi" button next to your Cube’s Spark Core, on the underside of the Cube. This will not delete any programs you have stored in the internal memory of the Spark Core, and just clears the WiFi credentials and puts the Spark Core back into WiFi Listening Mode. Then repeat the process above to connect to the new WiFi location.

Loading Cubetube Apps

CubeTube is in beta and available to the earliest Cube owners. Go to CubeTube.org and log in your Cube using the same user name and password you used to create an account on Spark.io or via the Spark App when you first connected your Cube to WiFi.

Then, when you click "Load App" on one of the CubeTube Apps, that App should automatically load onto your Cube, as long as your Cube is connected to WiFi.

Code for Cubes

Overview

There are two types of programs that can control Cubes. One is Native code, which lives entirely in the internal memory of the Cubes. The second is Streaming code, which requires a computer to stream content to your cube.

Native code does not require WiFi. Streaming code requires WiFi, but is more powerful in the sense that you can connect peripherals to your computer to interact with your Cube (such as the Leap Motion, MakeyMakeys, Kinects...).

Streaming is required when controlling multiple cubes.

Mode 1: Native Code

To get Native code Apps onto your Cube, it’s a one step process. Just log into CubeTube as described above and then click on any of the Native code apps and that app will download to your Cube.

Native code is written in Wiring, which is the fancy name for the Arduino programming language. You can write code on the Spark Web IDE.

Mode 2: Streaming Code

To get Streaming code working on your Cube, it’s a two step process.

Step 1: Log into CubeTube as described above and then load the Listener App onto your Cube.

Step 2 Download the Streaming code from any of the Streaming Apps to your computer. We’ve been running Streaming Apps in Processing, but folks are starting to write and stream python and Java Apps to their Cubes.

Make sure your computer is connected to WiFi, and then any Streaming Apps running on your computer should play in your Cube (loaded with Listener as described in 1.

If using Processing for your Streaming Apps, you’ll need to download the L3D Library. More details here: https://github.com/enjrolas/L3D-library

Alex has also put together a guide on designing graphics for your Cube in Streaming Mode here: http://www.instructables.com/id/How-to-Draw-Sweet-3D-Graphics-for-LED-cubes/

If you’d like faster streaming, try using a dedicated nano-router, like this one for about $20.
